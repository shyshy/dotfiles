# Dotfiles Modernization Design

## Goal

Restructure the dotfiles repo to use GNU Stow for portable config management across machines. Only track actively used tools. Ensure no secrets or environment-specific values are committed.

## Tools Managed

- zsh (zshrc + zimrc)
- neovim (LazyVim)
- zellij
- ghostty
- git (gitconfig + global ignore)

Starship is used but not configured (defaults) — no config file tracked. The `eval "$(starship init zsh)"` line in `.zshrc` is retained.

## Repo Structure

GNU Stow layout. Each top-level directory is a "package" mirroring the target path from `$HOME`:

```
dotfiles/
  README.md
  .gitignore
  .stow-local-ignore
  git/
    .gitconfig
    .config/git/
      ignore
  zsh/
    .zshrc
    .zimrc
  nvim/
    .config/nvim/
      init.lua
      lazyvim.json
      .neoconf.json
      stylua.toml
      lua/
        config/
          autocmds.lua
          keymaps.lua
          lazy.lua
          options.lua
        plugins/
          zellij-navigation.lua
  zellij/
    .config/zellij/
      config.kdl
  ghostty/
    .config/ghostty/
      config
```

Each package is stowed independently: `stow zsh`, `stow nvim`, etc.

## Files Removed

| File | Reason |
|---|---|
| `.vimrc` | Legacy Vim config with hardcoded `/Users/dshy/.cache/dein` path. Superseded by LazyVim. |
| `.tmux.conf` | Legacy tmux config. Superseded by zellij. |
| `.tmux.statusline` | Legacy tmux statusline. Superseded by zellij. |
| `init.lua` (repo root) | Standalone nvim config. Superseded by LazyVim setup at `~/.config/nvim/`. |
| `nvim/.config/nvim/lua/plugins/example.lua` | LazyVim boilerplate that no-ops (`if true then return {} end`). |

## Files Cleaned Up

### `.zshrc`

- Remove Arch-specific aliases (`pacman`, `hyprconf`) from active config
- Remove commented-out `. "$HOME/.local/bin/env"` line
- Add `[[ -f ~/.zshrc.local ]] && source ~/.zshrc.local` at end
- Add a comment above the local source line suggesting what goes there (OS-specific aliases, work env, secrets)

### `nvim/init.lua`

- Remove `-- Use system clipboard (Arch)` comment
- The file currently duplicates keymaps that should live in `lua/config/keymaps.lua`. Move custom keymaps there and leave `init.lua` as just the LazyVim bootstrap require.

### `zellij/config.kdl`

- Remove autogenerated header comment referencing machine-specific backup path (`/Users/ds/.config/zellij/config.kdl.bak.1`)

### `.gitconfig`

- Tracked version contains only portable settings:
  ```gitconfig
  [core]
    editor = nvim -f
  [include]
    path = ~/.gitconfig.local
  ```
- `user.name` and `user.email` go in `.gitconfig.local` (gitignored), since work machines use different identity

## Gitignored Items

```gitignore
# Machine-specific local overrides
.zshrc.local
.gitconfig.local

# Generated/binary files
**/lazy-lock.json
**/vim-zellij-navigator.wasm

# Stow internal
.stow-local-ignore
```

## `.stow-local-ignore`

Prevents stow from symlinking repo-level files that aren't config:

```
README.md
docs
.git
.gitignore
```

## Environment-Specific Config Pattern

Tracked configs are portable and environment-agnostic. Machine-specific values use the `.local` file pattern:

- `.zshrc` sources `~/.zshrc.local` if it exists
- `.gitconfig` includes `~/.gitconfig.local` if it exists
- Both `.local` files are gitignored

Examples of what goes in `.zshrc.local`:
- Arch: `alias pacman='sudo pacman'`
- macOS Homebrew PATH adjustments
- Work-specific env vars, tool configs
- API keys or tokens

## Nvim Git History

The `~/.config/nvim/` directory currently has its own `.git` from the LazyVim starter clone. This is removed — the files are absorbed directly into the dotfiles repo. The LazyVim starter history has no ongoing value.

Files from `~/.config/nvim/` NOT tracked:
- `.git/` (starter repo history)
- `.gitignore` (replaced by dotfiles repo `.gitignore`)
- `lazy-lock.json` (auto-generated lockfile)
- `lua/plugins/example.lua` (boilerplate)
- `LICENSE`, `README.md` (LazyVim starter docs)
