# Dotfiles Modernization Design

## Goal

Restructure the dotfiles repo to use GNU Stow for portable config management across machines. Only track actively used tools. Ensure no secrets or environment-specific values are committed.

## Tools Managed

- zsh (zshrc + zimrc)
- neovim (LazyVim)
- zellij
- ghostty
- git (gitconfig + global ignore)

Starship is used but not configured (defaults) — no config file tracked. The `eval "$(starship init zsh)"` line in `.zshrc` is retained.

## Repo Structure

GNU Stow layout. Each top-level directory is a "package" mirroring the target path from `$HOME`:

```
dotfiles/
  README.md
  .gitignore
  .stow-local-ignore
  docs/                             # specs and planning (not a Stow package)
  git/
    .gitconfig
    .config/git/
      ignore
  zsh/
    .zshrc
    .zimrc
  nvim/
    .config/nvim/
      init.lua
      lazyvim.json
      .neoconf.json
      stylua.toml
      lua/
        config/
          autocmds.lua
          keymaps.lua
          lazy.lua
          options.lua
        plugins/
          zellij-navigation.lua
  zellij/
    .config/zellij/
      config.kdl
  ghostty/
    .config/ghostty/
      config
```

Each package is stowed independently: `stow zsh`, `stow nvim`, etc.

## Files Removed

| File | Reason |
|---|---|
| `.vimrc` | Legacy Vim config with hardcoded `/Users/dshy/.cache/dein` path. Superseded by LazyVim. |
| `.tmux.conf` | Legacy tmux config. Superseded by zellij. |
| `.tmux.statusline` | Legacy tmux statusline. Superseded by zellij. |
| `init.lua` (repo root) | Standalone nvim config. Superseded by LazyVim setup at `~/.config/nvim/`. |
| `nvim/.config/nvim/lua/plugins/example.lua` | LazyVim boilerplate that no-ops (`if true then return {} end`). |

## Files Cleaned Up

### `zsh/.zshrc`

- Remove Arch-specific aliases (`pacman`, `hyprconf`) from active config
- Remove commented-out `. "$HOME/.local/bin/env"` line
- Retain `alias zshrc` and `alias vimrc` (reviewed — both point to valid paths after migration)
- Add `[[ -f ~/.zshrc.local ]] && source ~/.zshrc.local` at end
- Add a comment above the local source line suggesting what goes there (OS-specific aliases, work env, secrets)

### `nvim/.config/nvim/init.lua`

- Remove `-- Use system clipboard (Arch)` comment
- Move all custom keymaps to `lua/config/keymaps.lua` (currently empty). These are the 12 keymap definitions: `<Leader>s` (gsub), `<Space>` (nohlsearch), `<F2>` (toggle line numbers), `,s` (save), `,q` (save+quit), `<Leader>e` (edit in dir), `<Leader>v` (vsplit in dir), `<Leader>p` (hsplit in dir), `<Leader>c` (yank to clipboard)
- Move `vim.opt.clipboard = "unnamedplus"` to `lua/config/options.lua` (currently empty — it's an option, not a keymap)
- Leave `init.lua` as just the LazyVim bootstrap `require("config.lazy")`

### `zellij/.config/zellij/config.kdl`

- Remove autogenerated header comment referencing machine-specific backup path (`/Users/ds/.config/zellij/config.kdl.bak.1`)
- Note: backup files (`config.kdl.bak`, `config.kdl.bak.1`) on the machine are not tracked — they remain only on disk

### `git/.gitconfig`

- Tracked version contains only portable settings:
  ```gitconfig
  [core]
    editor = nvim -f
  [include]
    path = ~/.gitconfig.local
  ```
- `user.name` and `user.email` go in `.gitconfig.local` (gitignored), since work machines use different identity
- Note: git's `[include]` silently skips missing files, so stowing `.gitconfig` works on a fresh machine before `.gitconfig.local` is created

### `git/.config/git/ignore`

- Preserve existing content (`**/.claude/settings.local.json`)

## Gitignored Items

```gitignore
# Machine-specific local overrides
.zshrc.local
.gitconfig.local

# Generated/binary files
**/lazy-lock.json
**/vim-zellij-navigator.wasm

# Stow internal
.stow-local-ignore
```

## `.stow-local-ignore`

Prevents stow from symlinking repo-level files that aren't config:

```
README.md
docs
.git
.gitignore
.claude
```

## Environment-Specific Config Pattern

Tracked configs are portable and environment-agnostic. Machine-specific values use the `.local` file pattern:

- `.zshrc` sources `~/.zshrc.local` if it exists
- `.gitconfig` includes `~/.gitconfig.local` if it exists
- Both `.local` files are gitignored

Examples of what goes in `.zshrc.local`:
- Arch: `alias pacman='sudo pacman'`
- macOS Homebrew PATH adjustments
- Work-specific env vars, tool configs
- API keys or tokens

## Nvim Git History

The `~/.config/nvim/` directory currently has its own `.git` from the LazyVim starter clone. This is removed — the files are absorbed directly into the dotfiles repo. The LazyVim starter history has no ongoing value.

Files from `~/.config/nvim/` NOT tracked:
- `.git/` (starter repo history)
- `.gitignore` (replaced by dotfiles repo `.gitignore`)
- `lazy-lock.json` (auto-generated lockfile)
- `lua/plugins/example.lua` (boilerplate)
- `LICENSE`, `README.md` (LazyVim starter docs)

## Zellij Plugin

The `vim-zellij-navigator.wasm` binary (~1.1MB) is not tracked in the repo. It must be downloaded manually on each machine.

- Source: https://github.com/hiasr/vim-zellij-navigator/releases
- Target: `~/.config/zellij/plugins/vim-zellij-navigator.wasm`
- The README includes this download step.

## Migration (Existing Machine)

On the machine where configs currently exist as real files, the transition order is:

1. `git rm` the legacy files (`.vimrc`, `.tmux.conf`, `.tmux.statusline`, root `init.lua`)
2. Create the Stow directory structure and move/copy config files into it
3. Remove existing real config files from `$HOME` that will become symlinks (`~/.zshrc`, `~/.zimrc`, `~/.gitconfig`, etc.) — Stow refuses to overwrite non-symlink files
4. Remove the `.git` directory from `~/.config/nvim/` (absorbing into dotfiles repo)
5. Run `stow <package>` for each package to create symlinks
6. Create `.zshrc.local` and `.gitconfig.local` with machine-specific values

On a fresh machine, step 3 is unnecessary (no existing files to conflict with).
